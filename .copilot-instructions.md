# Copilot Instructions for pyifb

## Project Overview
This is `pyifb`, a Python package providing bindings for the ISO Fortran 2003 standard `ISO_Fortran_binding.h`. It allows Python code to interact with Fortran arrays and descriptors through the CFI (C Fortran Interface) specification.

## Key Technologies
- **Python C Extension**: Uses Python C API for creating extension modules
- **Limited API**: Restricted to stable Python C API functions (PY_LIMITED_API=0x030A0000)
- **ISO_Fortran_binding.h**: GCC's implementation of Fortran C interoperability
- **CFI Specification**: N1942.txt contains the ISO Fortran 2003 binding specification - use this document as primary reference for CFI types, functions, and behavior
- **CFI Types**: CFI_cdesc_t (C descriptor) and CFI_dim_t (dimension info)
- **Build System**: setuptools with C compilation

## Architecture
- `ifb.c`: Main C extension implementation
- `ifb.h`: Header with type definitions and compiler-specific macros
- Python types: `CFI_cdesc_t` and `CFI_dim_t` exposed as Python classes
- Serialization: `to_bytes()`/`from_bytes()` for passing descriptors

## Coding Conventions
- **Testing**: All new code must have comprehensive tests - add tests to appropriate test files before committing
- **Code Formatting**: Run black on any changed Python files: `black pyifb/__init__.py` or `black pyifb/`
- **Limited API Compliance**: Only use stable C API functions - avoid deprecated or unstable APIs
- **Error Handling**: Always return `NULL` on failure, set `PyErr_SetString()` appropriately
- **Reference Counting**: Use `Py_INCREF`/`Py_DECREF`, `Py_XDECREF` for cleanup
- **Memory Safety**: Validate input sizes, check for NULL pointers before operations
- **Type Safety**: Use appropriate Python C API types (`Py_ssize_t`, `size_t`, etc.)
- **Naming**: Functions prefixed with `PyCFI_`, variables descriptive (e.g., `dim_obj` not `tmp`)

## Common Patterns
- **Module Initialization**: Use `PyModule_AddObjectRef` for adding types to module
- **Type Creation**: `PyType_FromSpec` with slots for methods/members/getsets
- **Buffer Validation**: Check `PyBytes_Size` against expected sizes before `memcpy`
- **Deallocation**: Check `base_addr != NULL` before `CFI_deallocate`

## Security Considerations
- Prevent buffer overflows by validating byte array sizes
- Safe memory deallocation with NULL checks

## Compiler Support
- GCC: Full support with all CFI macros
- Intel ICX: Fallbacks for missing macros (-1 values)
- LLVM/Clang: Similar fallbacks

## Testing
- pytest-based test suite in `tests/`
- Covers type creation, serialization, Fortran integration
- Run with `python -m pytest tests/ -v`

## Build Commands
- `python setup.py build_ext --inplace`: Build extension in place
- `python -m pytest tests/`: Run tests
- `python setup.py sdist`: Create source distribution

## Important Notes
- **Documentation Reference**: N1942.txt in the workspace root contains the ISO Fortran 2003 binding specification - consult this document for authoritative information on CFI functions, types, and behavior
- CFI descriptors must be properly allocated/deallocated using CFI functions
- Rank-0 descriptors are scalars, higher ranks are arrays
- Dimension info stored in variable-sized array after base descriptor
- Use `CFI_SUCCESS` and error codes for status checking
- **Limited API**: Extension uses PY_LIMITED_API - ensure all C API calls are stable and available in the limited API</content>
